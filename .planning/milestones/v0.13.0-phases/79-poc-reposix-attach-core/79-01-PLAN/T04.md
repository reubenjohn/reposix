Strategy: prepare a local cache state with `version=1` for record id=1.

Outline:

```bash
#!/usr/bin/env bash
# POC path (c): cheap precheck on SoT mismatch.
set -euo pipefail
SCRIPT_DIR="$(cd -- "$(dirname -- "${BASH_SOURCE[0]}")" &> /dev/null && pwd)"
cd "${SCRIPT_DIR}"

echo "=== Path (c): cheap precheck on SoT mismatch ==="

# Start one sim.
cargo run -p reposix-sim --quiet -- --bind "127.0.0.1:${SIM_PORT}" --ephemeral --no-seed &
SIM_PID=$!
sleep 1

# Seed: record id=1, version=1.
curl -fsS -X POST "http://127.0.0.1:${SIM_PORT}/projects/demo/records" \
  -H 'Content-Type: application/json' \
  -d '{"id":1,"title":"v1","state":"open","body_md":"original"}'

# Snapshot the local cache view (record's version is "1").
LOCAL_VERSION=1

# SoT drifts: someone PATCHes the record server-side, bumping to version=2.
curl -fsS -X PATCH "http://127.0.0.1:${SIM_PORT}/projects/demo/records/1" \
  -H 'Content-Type: application/json' \
  -d '{"body_md":"server-side bump"}'

# Cheap precheck: GET the record, compare its version to LOCAL_VERSION.
SERVER_VERSION=$(curl -fsS "http://127.0.0.1:${SIM_PORT}/projects/demo/records/1" | python3 -c 'import json,sys;print(json.load(sys.stdin)["version"])')
echo "[POC c] local=${LOCAL_VERSION} server=${SERVER_VERSION}"

if [[ "${SERVER_VERSION}" != "${LOCAL_VERSION}" ]]; then
  echo "[POC c] MISMATCH: refusing fast — SoT drifted (local=${LOCAL_VERSION}, server=${SERVER_VERSION})"
  echo "[POC c] error refs/heads/main fetch first"
  echo "[POC c] hint: confluence has changes since your last fetch; git pull --rebase"
  echo "[POC c] (precheck completed without reading stdin or attempting any write)"
else
  echo "[POC c] UNEXPECTED: versions match"
fi

kill "${SIM_PID}" 2>/dev/null || true
```

The POC explicitly demonstrates that:

1. The version mismatch IS detectable via a single REST GET on the record.
2. The precheck does NOT need to enumerate the entire backend (the
   `list_changed_since` sketch in P81 is the production version of this;
   the POC uses a single-record GET to keep findings sharp).
3. The error message shape matches what the production bus remote will
   emit (per architecture-sketch step 3 hint text).
