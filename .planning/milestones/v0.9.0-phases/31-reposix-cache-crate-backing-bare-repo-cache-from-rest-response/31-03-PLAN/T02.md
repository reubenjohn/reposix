    Step 1 — Create `crates/reposix-cache/tests/compile-fail/untainted_new_is_pub_crate.rs`:
    ```rust
    //! This file MUST fail to compile.
    //!
    //! ARCH-02 relies on `Untainted::new` being `pub(crate)` in reposix-core
    //! so downstream crates cannot directly construct an untainted value and
    //! bypass the `sanitize` ritual. This fixture attempts the construction
    //! from outside `reposix-core` and expects a privacy error.

    fn main() {
        // This line MUST NOT compile: Untainted::new is pub(crate) inside
        // reposix-core; calling it from reposix-cache is a privacy error.
        let _u: reposix_core::Untainted<Vec<u8>> = reposix_core::Untainted::new(vec![1, 2, 3]);
    }
    ```

Splitting into two `#[test]` functions (each with its own `TestCases::new()`) runs them as separate test cases, which is fine — it just means two cargo-test lines in the output. Both patterns work; the split is clearer for diagnostic reporting.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Type system → runtime | Plan 03's guard is mechanical (compile-time). If the compile-fail fixtures silently start PASSING compilation (e.g. because a future contributor adds `impl From<Tainted<T>> for Untainted<T>`), the Tainted discipline is broken and the fixture should catch it. |
| Downstream crate → `reposix-core` privacy | Plan 03's second fixture locks `Untainted::new`'s `pub(crate)` visibility — a refactor that accidentally changes it to `pub` would let the fixture compile successfully, flagging the regression immediately. |
