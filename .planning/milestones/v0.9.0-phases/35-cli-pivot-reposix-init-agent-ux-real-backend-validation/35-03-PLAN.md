# Phase 35 Plan 03 — Real-backend integration tests

## Goal

The dark-factory regression test must be runnable against a real backend when creds are present, skipped cleanly when they are absent. Sim-only coverage does not satisfy ARCH-16. Per the phase prompt, ≥1 of {TokenWorld, reubenjohn/reposix, JIRA TEST} must run when creds present.

## Tasks

### Task 35-03-A: Real-backend test scaffolding

<read_first>
- `crates/reposix-confluence/tests/contract.rs` lines 61-74 (`skip_if_no_env!` macro source-of-truth).
- `crates/reposix-jira/tests/contract.rs` (JIRA env contracts).
- `35-RESEARCH.md` §R4.
</read_first>

<action>
1. Create `crates/reposix-cli/tests/agent_flow_real.rs` with three tests, all `#[ignore]`-gated:
   - `dark_factory_real_github` — `skip_if_no_env!("GITHUB_TOKEN")`. Runs `reposix init github::reubenjohn/reposix /tmp/df-gh-<pid>`. Asserts the local repo has the right config and that `git fetch` would work *if the helper supported real backends*. Per CLAUDE.md the helper currently hardcodes SimBackend (32-SUMMARY); a "soft" assertion is acceptable: the test verifies the *init* succeeds and the *url* is correct, then notes that fetch against real GitHub is deferred to a future phase.
   - `dark_factory_real_confluence` — `skip_if_no_env!("ATLASSIAN_API_KEY", "ATLASSIAN_EMAIL", "REPOSIX_CONFLUENCE_TENANT")`. Runs `reposix init confluence::TokenWorld /tmp/df-conf-<pid>`. Same soft-assertion as github.
   - `dark_factory_real_jira` — `skip_if_no_env!("JIRA_EMAIL", "JIRA_API_TOKEN", "REPOSIX_JIRA_INSTANCE")`. JIRA project key from `JIRA_TEST_PROJECT` or `REPOSIX_JIRA_PROJECT`, default `TEST`. Runs `reposix init jira::<key> /tmp/df-jira-<pid>`.
2. Each test uses the same `skip_if_no_env!` macro pattern (copy verbatim into the file per the existing convention).
3. Resolve the JIRA project key: `std::env::var("JIRA_TEST_PROJECT").or_else(|_| std::env::var("REPOSIX_JIRA_PROJECT")).unwrap_or_else(|_| "TEST".to_owned())`.
4. CI integration-contract job names (mirroring the existing pattern): `integration-contract-confluence-v09`, `integration-contract-github-v09`, `integration-contract-jira-v09`. Phase 36 wires the workflow file; we just write the tests.
</action>

<acceptance_criteria>
- Without env vars: each test skips cleanly with `SKIP: env vars unset:` stderr.

</acceptance_criteria>
