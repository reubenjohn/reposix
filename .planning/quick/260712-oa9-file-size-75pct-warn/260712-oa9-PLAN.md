# Quick Task 260712-oa9: 75% file-size early-warning tier

## Goal

Owner ask (STATE handover, 2026-07-12): a **75% file-size early-warning gate** — surface
files APPROACHING their progressive-disclosure ceiling before they blow past it, without
changing the existing block-later contract.

## Design (as implemented)

`structure/file-size-limits.sh` now sorts each in-scope tracked file into two tiers by
`pct = size*100/limit` (integer):

1. **75 ≤ pct < 100 — EARLY-WARNING (new).** Non-blocking WARN summary to stderr: a
   header naming the band count + "consider a progressive-disclosure split", then band
   files sorted by pct DESC (top-12, then `… and N more at ≥75%` when >12), each line
   `file — size/limit chars (PCT%)`. Print-only (no WARN verdict status in verdict.py;
   precedent: the pre-push timing tripwire). ALWAYS emitted, **independent of
   `--warn-only`/the waiver**, and NEVER affects the exit code.
2. **pct ≥ 100 (size > limit) — OVER-BUDGET (unchanged).** `exit 1` by default;
   `--warn-only` flips ONLY this tier to `exit 0`. The block-later contract (reactivates
   when the flag/waiver lapse on 2026-08-08) is byte-for-byte preserved.
