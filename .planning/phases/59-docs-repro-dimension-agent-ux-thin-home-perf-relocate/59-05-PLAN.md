**Python hyphen-vs-underscore decision tree (per pivot rule above):**

The test file MUST be able to import the production code. Two paths:

**Option A: keep hyphens, use runpy:**

```python
# In quality/gates/perf/test-bench-token-economy.py:
import runpy
from pathlib import Path
PROD = Path(__file__).parent / "bench-token-economy.py"
prod_module = runpy.run_path(str(PROD), run_name="bench_token_economy")
# Now prod_module is a dict of names from the production module:
some_function = prod_module["some_function"]
```

This works for invoking entire functions but is awkward for unit tests that need many imports.

**Option B: keep production file underscored:**

`scripts/bench_token_economy.py` -> `quality/gates/perf/bench_token_economy.py` (NOT bench-token-economy.py).
`scripts/test_bench_token_economy.py` -> `quality/gates/perf/test_bench_token_economy.py` (NOT test-bench-token-economy.py).

The test then imports normally:
```python
from bench_token_economy import some_function
```

**Wave E recommendation:** Option B. Rationale:
- Less work; preserves the existing test imports verbatim.
- The hyphen convention in `quality/gates/release/` etc. is _shell scripts_ (.sh files); Python files in the repo predominantly use underscores per PEP8.
- Survey the existing codebase: `find quality/gates -name '*.py'` -> P57/P58 shipped `quality/gates/structure/freshness-invariants.py`, `quality/gates/release/installer-asset-bytes.py`, `quality/gates/code/check-fixtures.py`. So the Python files DO use hyphens. But they are entry-point scripts invoked via path, not imported as modules. The test scenario is unique to the perf dimension because the test imports the production code.

**Final recommendation:** Option B for the perf-dimension Python files (preserves test imports + matches PEP8 + the existing hyphenated `.py` files in other dimensions are NOT imported anywhere, so they don't face this constraint). Document this decision in the wave summary so future P63 SIMPLIFY-12 understands why these two files don't follow the hyphen pattern.
