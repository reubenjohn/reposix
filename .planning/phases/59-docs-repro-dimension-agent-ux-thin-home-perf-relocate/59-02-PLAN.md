**Fenced block extraction algorithm (use a tested approach; do not over-engineer):**

Markdown fenced blocks are delimited by ```` ``` ```` (three backticks) optionally followed by a language tag. The simplest correct extractor:

```python
FENCE_RE = re.compile(r'^```(\w*)\s*$', re.MULTILINE)

def extract_blocks(path: Path) -> list[dict]:
    text = path.read_text(encoding="utf-8")
    blocks = []
    in_block = False
    block_start = 0
    block_lang = ""
    block_content_lines: list[str] = []
    for line_no, line in enumerate(text.splitlines(), start=1):
        m = FENCE_RE.match(line)
        if m and not in_block:
            in_block = True
            block_start = line_no
            block_lang = m.group(1)
            block_content_lines = []
        elif m and in_block:
            in_block = False
            content = "\n".join(block_content_lines)
            blocks.append({
                "file": str(path),
                "start_line": block_start,
                "end_line": line_no,
                "lang": block_lang,
                "content": content,
                "sha256": hashlib.sha256(content.encode("utf-8")).hexdigest()[:16],
                "derived_id": f"snippet/{path}:{block_start}-{line_no}",
            })
        elif in_block:
            block_content_lines.append(line)
    return blocks
```

This handles the typical case. Edge cases (nested fences, indented fences in lists) are acceptable false-positives for v0.12.0 — the catalog operator can write `--write-template` rows for genuine cases and the allow-list (if it ships) covers the rest.
