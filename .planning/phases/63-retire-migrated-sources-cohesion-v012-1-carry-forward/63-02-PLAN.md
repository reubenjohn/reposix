<objective>
Wave 2 -- the SIMPLIFY-12 audit + MIGRATE-01 retirement wave. Per the SIMPLIFY-12 contract: `find scripts/ -maxdepth 1 -type f | grep -v hooks | grep -v install-hooks.sh` returns empty, OR every remaining file has an explicit `quality/catalogs/orphan-scripts.json` waiver row.

This wave does the actual caller-scan + per-script decision + retirement work. The Wave 1 contract (catalog schema + verifier path locked) is the input; the output is (a) deletions where safe, (b) waiver rows where shim survives, (c) the audit document recording every decision, (d) the orphan-scripts-audit.py verifier so Wave 6 can re-grade the rows mechanically.

Process per script (top-level only, hooks/install-hooks excluded per SIMPLIFY-12 carve-out):
1. `git grep -nE '\\b<basename>\\b' -- ':!.planning/archive' ':!quality/SURPRISES-archive-*.md' ':!scripts/<basename>'` -- get caller list.
2. Categorize callers: code (cargo / yaml / py / sh / makefile / justfile), docs (md / rst), planning (.planning/**), comments-only.
3. Decide:
   - **DELETE** if zero non-doc / non-self / non-archive callers AND a canonical drop-in exists AND no CLAUDE.md section names the script path as user-facing.
   - **SHIM-WAIVED** if the script is already a thin shim AND callers exist OR CLAUDE.md/docs reference the path AND OP-5 reversibility argues for keeping it (pre-push hook stability, dev-loop muscle memory).
   - **KEEP-AS-CANONICAL** if the script has no canonical home in quality/gates/<dim>/ (e.g., scripts/p56-* are themselves the canonical install-evidence rehearsal pattern).
4. Record in `quality/reports/audits/scripts-retirement-p63.md` (per-script section).
5. Apply: `git rm` for DELETE; append row to `quality/catalogs/orphan-scripts.json` for SHIM-WAIVED + KEEP-AS-CANONICAL.

Outputs: the audit doc, the populated orphan-scripts.json, the new verifier script, deletions in scripts/.
</objective>
