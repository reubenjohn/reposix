<serialization>
Every wave is ONE atomic executor lane. Hard constraints (root + crates CLAUDE.md):
- **One tree-writer + one cargo invocation machine-wide at a time — strictly serial.** W2
  and W3 both edit `doc_alignment.rs` and both compile `reposix-quality`; run them one
  after the other, never concurrently. Cargo is FOREGROUND-only, never backgrounded.
- **Leaf isolation:** any `reposix`/`git config`/sim-seed test setup runs in a throwaway
  `/tmp` clone, `cd`-ing in the SAME Bash invocation (`.claude/hooks/leaf-isolation-guard.sh`).
  W1/W3/W5 verifications that shell out to `reposix-quality doc-alignment walk` use the
  `quality/gates/docs-alignment/walk.sh` wrapper (grades a /tmp copy — never mutates the
  committed catalog).
- **Commit before ending any turn** (uncommitted = didn't happen). Per-wave atomic commits
  are enumerated in each task's action.

════════ VERBATIM — SendMessage tier limitation (do not paraphrase) ════════
SendMessage is not granted at the phase-coordinator (C2) tier or below; L0→C2 and C2→main
work, C2→child and child→C2 fail; therefore C2-tier coordinators serialize strictly and
close phases via FRESH verifier→executor LEAVES, never fork-to-resume.
════════════════════════════════════════════════════════════════════════════

Consequence for this phase: the coordinator dispatches W1, waits for the leaf to return,
dispatches W2, and so on — no mid-lane hand-back. A blocking finding (e.g. W5 confirm-retire
guard, below) returns as a checkpoint to the coordinator, which opens a fresh leaf.
</serialization>
