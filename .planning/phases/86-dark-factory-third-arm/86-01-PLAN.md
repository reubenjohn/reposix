## Strategy: shell-stub (no real LLM in CI)

Spawning a real subagent in CI is heavyweight and brittle. Instead the third arm uses the **shell-stub approach**:

1. Emulate the agent's expected git workflow (vanilla `git clone` of a file:// mirror, `reposix attach <bus-url>`, edit, `git push <reposix-remote> main`).
2. Assert the helper's stderr / `--help` output at each step contains the **teaching strings** the agent would learn from.

The "agent recovers from stderr" property is then proved by stderr containing the recovery hints, not by actually running an LLM. This matches CLAUDE.md OP-1 (verify against reality — committed artifacts) AND keeps the sim arm CI-fast (target ≤ 60s wall time).

## Sim coverage: PASS (this phase). TokenWorld arm: substrate-gap-deferred per P84.

The sim arm closes the falsifiable threshold for P86. The TokenWorld arm is stub-blocked behind a `--real-tokenworld` flag with documented gating (citing the P84 SURPRISES-INTAKE entry). Reasoning:

- P84 surfaced that the live workflow on the GH mirror cannot install `reposix-cli v0.12.0` (yanked-gix + missing binstall artifacts).
- Until v0.13.x ships to crates.io with working binstall substrate, the TokenWorld round-trip cannot run end-to-end.
- The sim arm exercises the full DVCS topology in-process (sim::demo as SoT, file:// bare repo as mirror) so the helper's teaching contract is fully covered today.
