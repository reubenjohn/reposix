## Risk + mitigation

- **Range-merge bug double-counts overlapping ranges.** Mitigation: `merge_ranges` is the single load-bearing helper; ≥4 dedicated unit tests covering overlap / adjacent / disjoint / single.
- **Eligible-set drift between coverage and plan-backfill.** Mitigation: explicit constant set in `eligible_files()` with comment naming `collect_backfill_inputs` as the parallel; if it drifts, both are wrong together (single point of update).
- **Live walker run mutates production catalog.** Mitigation: walker run is the EXPECTED operation in this phase — it captures the measured baseline. Catalog mutation is a normal commit artifact, not a side-effect.
- **Workspace-wide cargo OOMs.** Mitigation: per-CLAUDE.md memory budget § — workspace-wide runs ONCE in Commit D, sequentially, no parallelization, after per-crate gates already clean.
- **Banned-words lint fails on new strings.** Mitigation: explicit sweep before each commit; "replace" is the well-known trap.
