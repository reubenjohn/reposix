<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| subprocess/worktree → shared source tree | A `reposix init` reaching the shared checkout via a subprocess/worktree bypasses the PreToolUse Bash-tool guard; only a binary-side refusal cuts this vector. |
| user-supplied target path → filesystem | The target path is attacker-influenceable (symlink/`..`/cd-back smuggling); canonicalization must resolve it soundly before the /tmp-safe decision. |
| init error stderr → operator | error strings are an exfiltration leg; carry no remote byte/credential (all `&'static`/local path). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-122-01 | Tampering | reposix init → shared tree | mitigate | Binary-side refusal (RPX-0406) when the canonical target nests inside a non-/tmp git tree, fired pre-mutation; worktree-shared-config self-check aborts before the config writes. |
| T-122-02 | Spoofing/Tampering | target path canonicalization | mitigate | Canonicalize the deepest existing ancestor (realpath -m semantics) so symlink / `..` / cd-back cannot smuggle a shared-tree op past the /tmp-safe check — mirrors leaf-isolation-guard.sh is_safe; test (c) proves it. |
| T-122-04 | Information disclosure | RPX-0406 error string | mitigate | Error is `&'static`/local-path only; no origin URL or credential interpolated. |
| T-122-08 | Denial of service (false refusal) | /tmp + attach flows | mitigate | Test (b) proves /tmp init still succeeds; test (d) proves attach adoption is un-regressed — the refusal is init-only and /tmp-safe. |
</threat_model>
