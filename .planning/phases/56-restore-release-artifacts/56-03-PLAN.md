**This is a known release-plz behaviour.** Document the implication in the release-fire-evidence.md (Task 56-03-G): the `releases/latest/download/...` pattern only works for the curl path when the cli release is genuinely the most recent. This is a v0.12.0 known-issue → MIGRATE-03 carry-forward to v0.12.1: investigate making release-plz tag the cli last, OR have release.yml's `gh release create` use `--latest` flag explicitly to override the natural pointer.
