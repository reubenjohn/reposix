**Current `release.yml` shape (key facts the executor needs without re-reading the whole file):**

1. `on.push.tags: ['v*']` — single-glob trigger (line 33-35). This is the bug.
2. The `plan` job's `Resolve tag + version` step (lines 51-64) strips `v` from the tag to produce `version`. With raw tag `reposix-cli-v0.11.3`, `${TAG#v}` returns `reposix-cli-v0.11.3` literally — no strip happens because the tag doesn't START with `v`. This is the second bug.
3. The `Stage archive` step (line 145) builds `STAGE="reposix-${TAG}-${{ matrix.target }}"`. With `TAG=reposix-cli-v0.11.3`, the stage dir + archive becomes `reposix-reposix-cli-v0.11.3-x86_64-unknown-linux-musl.tar.gz` — not what docs/installers expect. The installer script (lines 240-243) constructs the URL `reposix-${TAG}-${target}.tar.gz`, so it would fetch the same wrong-prefix path. **They match each other**, but they DON'T match the documented URLs in CHANGELOG/index.md/README.md (which use `reposix-vX.Y.Z-…`).
4. The `host` job's CHANGELOG-extraction step (line 296) uses `awk -v tag="## [${TAG}]"` to find the section in CHANGELOG.md. CHANGELOG entries are headed `## [v0.11.0] — 2026-04-26` — they use the `v*` form, not `reposix-cli-v*`. So with raw tag `reposix-cli-v0.11.3`, the awk lookup fails silently (the `body.md` falls back to the default "see CHANGELOG.md" line — not catastrophic but a separate friction).
5. The `upload-homebrew-formula` job's `Generate + commit formula` step (lines 384-426) hardcodes `reposix-${TAG}` archive names. Same fix shape as #3.
6. The `gh release create` step (line 336) uses `--title "reposix ${TAG}"`. With `TAG=reposix-cli-v0.11.3` the GH-release title becomes "reposix reposix-cli-v0.11.3" — ugly but not breaking.

**Number of `${TAG}` references in release.yml that need to become `v${VERSION}` (or `${RELEASE_TAG}` where the literal release name is needed):** approximately 8 substitutions — the `Stage archive` step, the installer-script `archive=` line for both `.sh` and `.ps1`, the CHANGELOG awk, the `gh release create --title`, the homebrew-formula `url` lines (3x for x86_darwin/arm_darwin/x86_linux), and the homebrew-formula awk-shasum-extract patterns.

**The decision tree:**
- If the count of substitutions is ≤ 8 (the case as analyzed above) AND every one is a clean `${TAG} → v${VERSION}` swap — Option A is correct.
- If during edit a substitution can't be cleanly mapped (e.g., a step that genuinely needs the full `reposix-cli-v0.11.3` git-tag-name to look up a git ref) — Option A still works for that step (use `${RELEASE_TAG}` as a separate output that always equals the actual git tag).
- If the GH-release name is required to remain `reposix-cli-v0.11.3` (which it is — release-plz creates it that way) but archive filenames need `v0.11.3`, the workflow needs **two** outputs: `release_tag` (the git ref name) and `version` (the cleaned version string).

**release-plz.yml is one-job, ~25 lines.** The Option B mirror-tag would add a step calling `git tag` + `git push origin` after release-plz's job completes successfully, gated on `contains(steps.release-plz.outputs.releases_created, 'reposix-cli')`.

```yaml
      - name: Mirror workspace tag for release.yml
        if: contains(steps.release-plz.outputs.releases_created, 'reposix-cli')
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          set -euo pipefail
          # When release-plz publishes reposix-cli, push a workspace-wide
          # vX.Y.Z tag so .github/workflows/release.yml (which fires on
          # `v*`) builds and uploads the multi-platform archives. Without
          # this mirror, only release-plz's per-crate `reposix-cli-v*` tag
          # exists, and release.yml does not fire.
          VERSION=$(jq -r '.releases[]
            | select(.package_name == "reposix-cli") | .version' \
            <<< '${{ steps.release-plz.outputs.releases }}')
          if [[ -z "$VERSION" || "$VERSION" == "null" ]]; then
            echo "could not extract reposix-cli version from release-plz outputs; skipping mirror tag"
            exit 0
          fi
          MIRROR_TAG="v${VERSION}"
          if git rev-parse "$MIRROR_TAG" >/dev/null 2>&1; then
            echo "tag ${MIRROR_TAG} already exists; skipping"
            exit 0
          fi
          git config user.name "reposix-release-bot"
          git config user.email "release-bot@users.noreply.github.com"
          git tag "$MIRROR_TAG"
          git push origin "$MIRROR_TAG"
          echo "pushed mirror tag ${MIRROR_TAG}; release.yml should fire automatically"
```
