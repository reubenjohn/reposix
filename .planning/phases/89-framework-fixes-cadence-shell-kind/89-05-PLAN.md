2. **Create `quality/gates/structure/deferral-pointer-linter.sh`** (mirrors RESEARCH § "Implementation Patterns" RBF-FW-05) — **REVIEW-FIX HIGH-2 woven in: every deferral-pattern match must produce ≥1 extracted phase number; zero extractions BLOCK explicitly**:
   ```bash
   #!/usr/bin/env bash
   # quality/gates/structure/deferral-pointer-linter.sh — RBF-FW-05
   # Greps crates/ for three deferral-pointer regex patterns and BLOCKs if
   # the named phase number has no .planning/phases/N-*/PLAN*.md (the deferred
   # work has nowhere to land — substrate-gap-deferred-but-no-substrate).
   #
   # ─────────────────────────────────────────────────────────────────────────
   # REVIEW-FIX HIGH-2: a deferral-pattern match that yields ZERO extracted
   # phase numbers (e.g., a comment `// substrate-gap-deferred` with no PNN
   # suffix) ALSO BLOCKs — the previous version of this script silently passed
   # such cases because the inner extraction loop never executed. The linter's
   # contract is "every deferral pointer cross-references a real downstream
   # PLAN", which a no-PNN deferral cannot satisfy. Three of the three current
   # deferral patterns DO require a P\d+ suffix; the BLOCK is for FUTURE drift
   # where a developer adds a bare `// substrate-gap-deferred` without naming
   # a phase.
   # ─────────────────────────────────────────────────────────────────────────
   #
   # Existence-only check per CONTEXT D-05c "Default for P89"; content
   # cross-reference is a P90/P95 polish.
   #
   # Implements catalog row structure/deferral-pointer-linter.
   set -euo pipefail
   SCRIPT_DIR="$(cd -- "$(dirname -- "${BASH_SOURCE[0]}")" &> /dev/null && pwd)"
   REPO_ROOT="$(cd "${SCRIPT_DIR}/../../.." &> /dev/null && pwd)"
   cd "$REPO_ROOT"

   PATTERNS=(
     'not yet wired in P[0-9]+'
     'lands? (alongside|in) P[0-9]+'
     'substrate-gap-deferred'
   )

   violations=0
   total_matches=0

   for pat in "${PATTERNS[@]}"; do
     while IFS=: read -r path lineno content; do
       total_matches=$((total_matches + 1))

       # Extract phase number(s) from the matched line content.
       # REVIEW-FIX HIGH-2: count extractions — if zero, BLOCK explicitly.
       extracted_phases=$(echo "$content" | grep -oE 'P[0-9]+' | sed 's/^P//' | sort -u)
       extracted_count=$(echo "$extracted_phases" | grep -c '^[0-9]' || true)

       if [[ "$extracted_count" -eq 0 ]]; then
         # Bare deferral pattern with NO phase number — silent-pass false
         # negative the previous version had. Block explicitly.
         printf '✖ %s:%s deferral pattern matched but no PNN suffix found\n' \
           "$path" "$lineno" >&2
         printf '   matched-line: %s\n' "$content" >&2
         printf '   pattern: %s\n' "$pat" >&2
         printf '   owner_hint: append the target phase number (e.g. `... until P95 lands`) OR scrub the deferral pointer\n' >&2
         violations=1
         continue
       fi

       # ≥1 phase extracted — verify each resolves to an existing PLAN file.
       while IFS= read -r N; do
         [[ -z "$N" ]] && continue
         if ! ls .planning/phases/${N}-*/*PLAN*.md >/dev/null 2>&1; then
           printf '✖ %s:%s references P%s but no .planning/phases/%s-*/*PLAN*.md exists\n' \
             "$path" "$lineno" "$N" "$N" >&2
           printf '   matched-line: %s\n' "$content" >&2
           violations=1
         fi
       done <<< "$extracted_phases"
     done < <(grep -rnHE "$pat" crates/ 2>/dev/null || true)
   done

   if [[ $violations -eq 1 ]]; then
     echo "" >&2
     echo "owner_hint: create the named phase's PLAN file at .planning/phases/N-*/N-NN-PLAN.md" >&2
     echo "            OR scrub the deferral pointer if the deferred work shipped" >&2
     echo "            OR append the missing PNN suffix if the pattern is bare" >&2
     echo "see: quality/catalogs/freshness-invariants.json row structure/deferral-pointer-linter" >&2
     exit 1
   fi
   echo "PASS: ${total_matches} deferral-pointer match(es) in crates/ all resolve to existing phase dirs with PLAN files (and all carry PNN suffixes)"
   ```

## Watchouts

- **Phase number renumbering risk** (RESEARCH § Risks → False-positive risk): if v0.13.0 phases get renumbered (unlikely; phase numbers are immutable post-shipping), existing references would break en masse. Document in CLAUDE.md (89-08) — do NOT add the doc here.
- **`grep -oE 'P[0-9]+'` extracts ALL phase numbers on a line** — a comment like `// P79-02 scaffold per P82+ rebase` matches both `P79` and `P82`. Both are checked for `.planning/phases/{79,82}-*/*PLAN*.md`. Either being missing BLOCKs.
- **`sort -u`** in the inner pipeline prevents duplicate processing if the same `P\d+` appears twice on the same line.
- **`ls ...glob/*PLAN*.md`** uses shell globbing — if zero matches, exits non-zero (suppressed by `>/dev/null 2>&1`); the negation `! ls` correctly captures absence. Sub-subagent verifies behavior with `set -x` if uncertain.
- **In-task expected behavior is PASS** — do NOT silence the linter or add allowlist markers. The two existing matches are intentional pointers to phases that DO exist; if either pointed to a phase without a PLAN file, the linter would correctly BLOCK and the planner would route the issue to that phase's authoring task.
- **REVIEW-FIX HIGH-2 — `extracted_count` calculation:** the `grep -c '^[0-9]'` against the `extracted_phases` variable expects each phase number on its own line (because of `sort -u` upstream). If `extracted_phases` is empty, `grep -c` returns 0 (with `|| true` to swallow grep's exit-1 on no-match). Sub-subagent verifies the count branch fires correctly with `set -x` against Scenario B before declaring done.
- **`while IFS= read -r N <<< "$extracted_phases"` vs `< <(echo "$extracted_phases")`:** the heredoc-string form (`<<<`) is used here because `extracted_phases` is already a captured string, not a stream. Both work; pick the form that matches the rest of the script style.
