# Plan 64-02: Crate skeleton + binary surface + hash binary + integration tests

## Goal

Land the entire `reposix-quality` Rust binary surface (umbrella + 9 doc-alignment verbs + run/verify/walk + hash_test_fn) with full unit and integration test coverage. After this plan, the binary is callable from the runner and from skill subagents.

## Steps

1. **Workspace plumbing.** Add `crates/reposix-quality` to workspace members. Verify needed deps exist at workspace level; add `syn = { version = "2", features = ["full"] }` and `quote = "1"` if missing. **Single `cargo check --workspace` to confirm — but only at the END of this plan, not mid-step (memory budget).**

2. **Crate skeleton commit (Commit A).** Create the crate, lib.rs, main.rs, catalog.rs, hash.rs, commands/{mod,run,doc_alignment}.rs (with stub bodies), src/bin/hash_test_fn.rs, the wrapper script, and the smoke + hash tests. `cargo test -p reposix-quality` PASS, clippy clean, fmt clean. Commit A.

3. **Subcommand bodies + golden tests commit (Commit B).** Implement each verb body, fill out the tests, run the full per-crate test suite. Catalog mutations are atomic (tmp + rename). Hash computation matches the standalone binary by sharing the hash module. Commit B.

4. **Final per-crate validation:** clippy + fmt + test all clean.

## Out of scope (later waves)

- Hook wiring (Wave 3 / plan 64-03)
- PROTOCOL.md / CLAUDE.md updates (Wave 3 / plan 64-03)
- Verifier dispatch (Wave 3 / plan 64-03)

## Risk + mitigation

- **`syn` parse fragility on impl blocks** → use `syn::visit::Visit` to walk all `ItemFn` and `ImplItemFn` nodes; collect by name; error if zero or >1 matches with the same simple name (the bind rationale must then qualify with `Self::method` form).
- **Cargo memory pressure** → strict per-crate cadence; `cargo build --jobs 2` if `cargo test -p` hits OOM.
- **Tests writing to the real `quality/catalogs/doc-alignment.json`** → use tempdirs in tests; pass catalog path via `--catalog <path>` flag (default = `quality/catalogs/doc-alignment.json`). The catalog path flag is added as a global `reposix-quality` flag.
