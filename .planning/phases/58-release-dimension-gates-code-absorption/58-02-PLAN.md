**Runner contract for verifier output** (from quality/runners/run.py run_row()):

The runner expects each verifier to:
1. Write a JSON file at the catalog row's `artifact` path (the verifier infers the path from its args or hardcoded mapping).
2. The artifact JSON must have `asserts_passed: [...]` and `asserts_failed: [...]` arrays. The runner reads these for verdict generation.
3. Exit 0 / 1 / 2 / timeout. The runner maps these to PASS / FAIL / PARTIAL / FAIL.

**Common verifier scaffolding:**

```python
#!/usr/bin/env python3
"""Quality Gates verifier — <name>."""
from __future__ import annotations
import argparse, json, sys, urllib.request, urllib.error
from datetime import datetime, timezone
from pathlib import Path

REPO_ROOT = Path(__file__).resolve().parents[3]
ARTIFACT_DIR = REPO_ROOT / "quality" / "reports" / "verifications" / "release"

def now_iso() -> str:
    return datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")

def http_get_json(url: str, timeout: int = 30):
    req = urllib.request.Request(url, headers={"User-Agent": "reposix-quality-gates/0.12.0"})
    try:
        with urllib.request.urlopen(req, timeout=timeout) as resp:
            return resp.status, json.loads(resp.read().decode("utf-8")), ""
    except urllib.error.HTTPError as e:
        return e.code, None, f"HTTPError: {e}"
    except (urllib.error.URLError, json.JSONDecodeError, OSError) as e:
        return 0, None, f"{type(e).__name__}: {e}"

def write_artifact(row_id: str, filename: str, asserts_passed, asserts_failed, extra=None):
    artifact = {"ts": now_iso(), "row_id": row_id,
                "asserts_passed": asserts_passed, "asserts_failed": asserts_failed}
    if extra: artifact.update(extra)
    out = ARTIFACT_DIR / filename
    out.parent.mkdir(parents=True, exist_ok=True)
    out.write_text(json.dumps(artifact, indent=2) + "\n", encoding="utf-8")
```

**Verifier-specific implementation notes:**

### gh-assets-present.py

Default `--tag latest` -> https://api.github.com/repos/<repo>/releases/latest. Specific tag -> https://api.github.com/repos/<repo>/releases/tags/<tag>. Asserts: HTTP 200; response.assets is a list with len >= min-assets; every asset has state == "uploaded".

### installer-asset-bytes.py

Mode A (asset-existence): HEAD --url; assert HTTP 200; assert content-length header (or content-length from a follow-up GET if HEAD lacks it) >= min-bytes. If --magic provided: GET first 32 bytes of url; assert leading bytes start with the prefix string.

GitHub redirects (releases/latest/download/...) return 302 -> releases/download/<tag>/.... urllib follows redirects by default. HEAD on redirect target may return content-length; if not, use GET range request "Range: bytes=0-31" to fetch the magic prefix.

Mode B (CI-job-status): runs `gh run list --workflow=<workflow-file> --branch=<branch> --status=success --limit=1 --json databaseId,conclusion`. Asserts subprocess exit 0; output contains a databaseId.

Mode detection: --url and --ci-check are mutually exclusive (assert exactly one is set).

Artifact filename mapping by URL pattern:
- url contains "reposix-installer.sh" -> install-curl-installer-sh.json (row install/curl-installer-sh)
- url contains "reposix-installer.ps1" -> install-powershell-installer-ps1.json (row install/powershell-installer-ps1)
- --ci-check mode -> install-build-from-source.json (row install/build-from-source)

### brew-formula-current.py

Hardcoded URL: https://api.github.com/repos/reubenjohn/homebrew-reposix/contents/Formula/reposix.rb. GET -> assert HTTP 200; response.content is base64-encoded. Decode -> formula text. Grep formula_text for `version "X.Y.Z"` (regex `version "(\d+\.\d+\.\d+)"`). GET https://crates.io/api/v1/crates/reposix-cli -> assert HTTP 200; extract response['crate']['max_version']. Assert formula version == crates.io max_version.

Writes TWO artifacts: brew-formula-current.json (row release/brew-formula-current) + install-homebrew.json (row install/homebrew).

### crates-io-max-version.py

Read Cargo.toml at REPO_ROOT, parse [workspace.package].version using tomllib (Python 3.11+):

```python
import tomllib
with open(REPO_ROOT / "Cargo.toml", "rb") as f:
    data = tomllib.load(f)
version = data["workspace"]["package"]["version"]
```

GET https://crates.io/api/v1/crates/<crate> with User-Agent header (crates.io requires UA). Assert HTTP 200; extract response['crate']['max_version']. Assert max_version == workspace version. Sleep 1s at end (rate-limit safety).

Artifact: crates-io-max-version-<crate>.json (e.g., crates-io-max-version-reposix-cli.json).

### cargo-binstall-resolves.py

Run `cargo binstall --no-confirm --dry-run <crate>` via subprocess.run([..., shell=False], capture_output=True, timeout=600).

Drop the --no-discover-github-token flag if cargo-binstall version is older than ~0.20+ (use try-once + retry pattern).

Parse stdout/stderr for URL pattern `github.com/reubenjohn/reposix/releases/download/`. If matched: exit 0 (PASS — prebuilt binary resolved). If stdout/stderr contains "Falling back to source" or "Falling back to install via 'cargo install'": exit 2 (PARTIAL — documented expected state per P56 SURPRISES.md row 3). If subprocess exit != 0 AND no source-fallback signal: exit 1 (FAIL). If cargo-binstall is not installed (FileNotFoundError): exit 1 with "cargo-binstall not installed in environment" — acceptable for local Wave B testing.

### __init__.py

Empty file. Python package marker.

### README.md (<=50 lines)

GFM table mapping verifier -> catalog rows -> cadence + a conventions paragraph + cross-references.
