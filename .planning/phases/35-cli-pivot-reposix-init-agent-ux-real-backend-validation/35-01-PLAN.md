# Phase 35 Plan 01 — `reposix init` subcommand replaces `reposix mount`

## Goal

Add `reposix init <backend>::<project> <path>` and replace `reposix mount` with a one-line migration error. Document the breaking change in CHANGELOG `[Unreleased]`. Update README to show the new flow.

## Tasks

### Task 35-01-A: Add `Init` subcommand + replace `Mount` body

<read_first>
- `crates/reposix-cli/src/main.rs` (existing clap dispatch)
- `crates/reposix-cli/src/mount.rs` (current mount handler)
- `crates/reposix-core/src/remote.rs` (URL shape: `reposix::<scheme>://<host>/projects/<slug>`)
- `35-RESEARCH.md` §R1 (translation table)
</read_first>

<action>
1. Create `crates/reposix-cli/src/init.rs` with a `pub fn run(spec: &str, path: PathBuf) -> Result<()>` that:
   - Parses `spec` as `<backend>::<project>` (split on first `::`).
   - Translates to a `reposix::<scheme>://<host>/projects/<project>` URL per R1.
     - `sim::<slug>` → `reposix::http://127.0.0.1:7878/projects/<slug>`.
     - `github::<owner>/<repo>` → `reposix::https://api.github.com/projects/<owner>/<repo>`.
     - `confluence::<space>` → uses `REPOSIX_CONFLUENCE_TENANT` env var.
     - `jira::<key>` → uses `REPOSIX_JIRA_INSTANCE` env var.
   - Runs the six-step git sequence via `std::process::Command`. The final `git fetch` step is best-effort: log on failure but don't bail (the user may not have creds yet).
2. Add `Init { spec: String, path: PathBuf }` variant to `Cmd` enum in `main.rs`. Wire dispatch.
3. Replace the `Mount::run` body (`crates/reposix-cli/src/mount.rs`) with a single-line `bail!` returning the migration message exactly as locked in CONTEXT decisions. Keep the clap variant + flags so `--help` still lists `mount` (so old scripts get a clear error rather than "unknown subcommand"). The fuse spawn code remains; deletion is Phase 36's job per the prompt.
4. Per CLAUDE.md `#![forbid(unsafe_code)]` and `#![warn(clippy::pedantic)]`. All public items doc-commented; `# Errors` sections on `Result`-returning fns.
</action>

<acceptance_criteria>
- `reposix init sim::demo /tmp/test-init` produces a directory where `git rev-parse --is-inside-work-tree` is `true` and `git config remote.origin.url` returns `reposix::http://127.0.0.1:7878/projects/demo`.
- `reposix mount /tmp/anything` exits non-zero (code 2) with stderr matching the locked migration message.
- `cargo clippy --workspace --all-targets -- -D warnings` clean.
- `cargo build --workspace` clean.
</acceptance_criteria>

<threat_model>
- Untrusted input: `<backend>::<project>` comes from the user. Validate the backend prefix is one of the four known values; reject others with a clear error.
- No shell injection: all args go through `std::process::Command` arg APIs, never `sh -c`.
- Frontmatter allowlist remains owned by the helper (Phase 34 mitigation), not affected here.
</threat_model>
