# Phase 35 Plan 04 — Latency benchmark + sales-asset doc + testing-targets

## Goal

Capture wall-clock latency per golden-path step against sim (mandatory) and each real backend (when creds present). Commit `docs/benchmarks/v0.9.0-latency.md` as a sales-asset Markdown table. Create `docs/reference/testing-targets.md` documenting the three sanctioned go-crazy-safe targets.

## Tasks

### Task 35-04-A: `scripts/v0.9.0-latency.sh` + supporting Rust harness

<read_first>
- `35-RESEARCH.md` §R3, §R5.
- `scripts/demo.sh` (style reference).
- `crates/reposix-cli/tests/cli.rs` (existing pattern for `assert_cmd`-driven integration tests).
</read_first>

<action>
1. Create `scripts/v0.9.0-latency.sh` that:
   - Spawns `reposix-sim --ephemeral` on a free port.
   - Runs the golden path (clone, first-blob fetch, sparse-batched checkout of 10 blobs, edit, push) timing each with `date +%s.%3N` deltas.
   - Emits Markdown rows to stdout in the format `| <step> | <sim ms> | <github> | <confluence> | <jira> |`. Empty cells where creds are absent.
   - Tail the script writes a fully-formatted Markdown file at `docs/benchmarks/v0.9.0-latency.md` (the script itself is the regenerator; running it overwrites the file).
   - Soft thresholds: emit `WARN:` lines if sim cold clone exceeds 500ms or any real-backend step exceeds 3s. Never `exit 1` on threshold breach.
2. `chmod +x scripts/v0.9.0-latency.sh`.
</action>

<acceptance_criteria>
- `bash scripts/v0.9.0-latency.sh` runs end-to-end against sim with exit 0.
- Output Markdown matches the format in §R5.
- Soft thresholds produce `WARN:` lines; do not fail the script.
</acceptance_criteria>

<threat_model>
- Sim binds to localhost; no external egress unless creds provided.
- Tempdirs are cleaned up by trap.
</threat_model>
