GREEN when all 6 items reach a recorded disposition and `main` stays GREEN.

## Task detail + discipline gates

### T1 — SIGKILL blast radius (HIGH, DP-2 prove-before-fix)

- **Do not blind-fix.** First build a committed minimal repro that drives a gate child to
  the kill path and asserts whether the signal propagates to the parent `run.py` process
  group (and whether an orphan `reposix sim` / port-7878 binding survives). Cross-ref the
  fd-inheritance sibling fix `cef3a2ea` for the reaping+isolation pattern.
- Fix `container-rehearse-sigkill-safe.sh` (and any sibling that `setsid`/group-kills)
  to target only its own child PID/subtree **only if** the repro confirms the mechanism
  AND a code-reviewer confirms it is mechanism-not-symptom.
- **Budget escape hatch:** if the repro is un-buildable within one lane budget, DOWNGRADE
  the entry to *suspicion* and keep it filed with the repro attempt recorded — do NOT ship
  a speculative kill-scoping change.
- Cargo/sim discipline: this is a P1 cargo/sim-dependent gate — **one cargo invocation
  machine-wide, FOREGROUND only**, leaf setup (sim seed / `reposix init` / `git`) in a
  `/tmp` clone with the `cd` in the SAME Bash invocation.

### T2 — hermetic-test-network-isolation stale mint (MEDIUM, honesty)

- Decide the branch: is the gate itself CI-broken (needs a portability fix so it denies
  the network deterministically in the CI sandbox), or is only the mint stale (the gate
  is legitimately environment-dependent and the committed PASS lies)?
- If code fix → prove-before-fix (reproduce the ~0.02s CI fast-fail locally under the
  sandbox's restriction before editing the gate).
- If mint-stale → honest re-mint: `NOT-VERIFIED` or `WAIVED + until_date` with a
  `skip_reason` naming the CI-sandbox restriction. Never leave a committed PASS the CI
  environment cannot reproduce (honesty dimension).

### T3 — shell-coverage counter-drift + F-K4b `--persist` friction (LOW-MED, DP-3)

- Run DP-3 (intake-design inversion) BEFORE writing code: is the right move to fix the
  drift, accept-FAIL, or decouple? The intake sketch lists all three; do not re-diagnose
  the drift a fourth time (cross-ref the two part-07 drift entries + `L1166`).
- **Row-level defusal is in-scope IFF** it is <1h + no-new-dep AND does not weaken the
  gate: drop the WARN-only assert #2 from `expected.asserts` so F-K4b congruence stops
  demoting on `--persist`, exactly mirroring `ba13553f`'s structurally-uncoverable-assert
  removal. Catalog-first: the row change is the artifact; a regression test locks it.
- **Explicitly deferred:** the deeper `scripts/shell_coverage.py` counting-mechanism
  reconciliation is **GTH-V15-99 → P128 Slot 2**. Do NOT open it here.
