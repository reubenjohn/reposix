**Why the runner-stdin-empty case?** When the pre-push hook (Wave E) invokes the runner, the runner discovers the cred-hygiene catalog row + invokes `bash quality/gates/structure/cred-hygiene.sh`. Wait — that's a problem. The cred-hygiene scan needs the push-ref ranges from stdin to be meaningful. If the runner doesn't pipe stdin, the scan is a no-op when invoked via the runner.

**Resolution:** The pre-push hook in Wave E does NOT delegate cred-hygiene to the runner. Instead:
- The pre-push hook receives stdin from git (the push-ref ranges).
- The hook BIFURCATES: (a) pipes stdin to `quality/gates/structure/cred-hygiene.sh` directly (preserves the scan); (b) AFTER cred-hygiene passes, invokes `quality/runners/run.py --cadence pre-push` for everything else.

Wave E lands the bifurcation; Wave D just provides the wrapper. The wrapper itself is correct: it scans stdin if present, no-ops if not.

This means the catalog row `structure/cred-hygiene` is INFORMATIONAL (the row tracks the contract; the actual scan happens in the pre-push hook bifurcation). The runner re-grades the row PASS when invoked because the wrapper exits 0 on empty stdin -- which is correct behavior (no ranges = no hits).

An alternative: the runner could pipe an empty `git rev-list --all --not --remotes` (no remote-equivalent comparison) into the cred-hygiene wrapper to scan the whole repo. That's overkill for pre-push (the hook context is "what's about to be pushed") and would slow the runner significantly. Keep the bifurcation: hook does cred-hygiene with stdin; runner does everything else.

#!/usr/bin/env bash
# quality/gates/code/cargo-fmt-check.sh -- code/cargo-fmt-check verifier (P60 Wave D).
#
# Wraps `cargo fmt --all -- --check` and writes a runner-readable artifact.
# Exits 0 = PASS (no formatting drift); 1 = FAIL (drift detected).
#
# Honors --row-id <id> (defaults to code/cargo-fmt-check) for catalog discrimination.

set -euo pipefail

readonly ROW_ID="${1:---row-id}"
readonly REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../../.." && pwd)"
readonly ARTIFACT_DIR="${REPO_ROOT}/quality/reports/verifications/code"
readonly ARTIFACT="${ARTIFACT_DIR}/cargo-fmt-check.json"

# Argparse: --row-id <id>
row_id="code/cargo-fmt-check"
if [[ "${1:-}" == "--row-id" && -n "${2:-}" ]]; then
  row_id="$2"
fi

cd "$REPO_ROOT"
mkdir -p "$ARTIFACT_DIR"

stdout=$(cargo fmt --all -- --check 2>&1) && exit_code=0 || exit_code=$?
ts=$(date -u +%Y-%m-%dT%H:%M:%SZ)

# Asserts: passed if exit_code==0; failed otherwise.
if [ "$exit_code" -eq 0 ]; then
  asserts_passed='["cargo fmt --all -- --check exits 0 (no formatting drift)"]'
  asserts_failed='[]'
else
  asserts_passed='[]'
  asserts_failed='["cargo fmt --all -- --check exit '${exit_code}' (drift detected; run cargo fmt --all)"]'
fi

# Encode stdout as JSON-safe (escape via jq -R or python).
stdout_json=$(printf '%s' "$stdout" | python3 -c "import sys,json; print(json.dumps(sys.stdin.read()))")

cat > "$ARTIFACT" <<EOF
{
  "ts": "$ts",
  "row_id": "$row_id",
  "exit_code": $exit_code,
  "stdout": $stdout_json,
  "stderr": "",
  "asserts_passed": $asserts_passed,
  "asserts_failed": $asserts_failed
}
EOF

if [ "$exit_code" -ne 0 ]; then
  echo "$stdout" >&2
fi
exit "$exit_code"
```

#!/usr/bin/env bash
# quality/gates/structure/cred-hygiene.sh -- structure/cred-hygiene verifier (P60 Wave D).
#
# Migrated VERBATIM from scripts/hooks/pre-push:41-122 lines (the
# credential-prefix scan body). The scan logic is a P0 security gate;
# any logic change without explicit threat-model review is silent
# downgrade per quality/PROTOCOL.md failure modes.
#
# Stdin: push-ref-range lines (matches the existing pre-push hook stdin
# contract). Empty stdin = no ranges to scan = exit 0 (correct).
# Non-empty stdin = iterate ranges, grep for credential prefixes.
#
# Exits 0 = PASS, 1 = FAIL (any credential prefix matched).
#
# Honors --row-id <id> (defaults to structure/cred-hygiene).

set -euo pipefail

readonly RED='\033[0;31m'
readonly GREEN='\033[0;32m'
readonly YELLOW='\033[1;33m'
readonly NC='\033[0m'

readonly REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/../../.." && pwd)"
readonly ARTIFACT_DIR="${REPO_ROOT}/quality/reports/verifications/structure"
readonly ARTIFACT="${ARTIFACT_DIR}/cred-hygiene.json"

row_id="structure/cred-hygiene"
if [[ "${1:-}" == "--row-id" && -n "${2:-}" ]]; then
  row_id="$2"
fi

cd "$REPO_ROOT"
mkdir -p "$ARTIFACT_DIR"

# VERBATIM PATTERNS + EXCLUDE_DIRS from scripts/hooks/pre-push:41-57
readonly PATTERNS=(
  'ATATT3[A-Za-z0-9_+/=-]{20,}'
  'Bearer[[:space:]]+ATATT3[A-Za-z0-9_+/=-]{20,}'
  'ghp_[A-Za-z0-9]{20,}'
  'github_pat_[A-Za-z0-9_]{20,}'
)
readonly EXCLUDE_DIRS=(
  '.git'
  'target'
  'node_modules'
  'scripts/hooks'
  'quality/gates/structure'  # this file itself
)

# Read stdin ref ranges (push-ref-line format from git).
scan_refs=()
while read -r local_ref local_sha remote_ref remote_sha; do
  if [[ -z "${local_sha:-}" ]]; then continue; fi
  if [[ "$local_sha" == "0000000000000000000000000000000000000000" ]]; then continue; fi
  if [[ "$remote_sha" == "0000000000000000000000000000000000000000" ]]; then
    base=$(git rev-list --max-parents=0 "$local_sha" 2>/dev/null | head -1)
    if [[ -n "$base" ]]; then scan_refs+=("${base}..${local_sha}"); else scan_refs+=("$local_sha"); fi
  else
    scan_refs+=("${remote_sha}..${local_sha}")
  fi
done

ts=$(date -u +%Y-%m-%dT%H:%M:%SZ)
hit=0
hits_log=""

if [ "${#scan_refs[@]}" -gt 0 ]; then
  for range in "${scan_refs[@]}"; do
    files=$(git diff --name-only "$range" 2>/dev/null || true)
    for file in $files; do
      skip=0
      for excl in "${EXCLUDE_DIRS[@]}"; do
        if [[ "$file" == "$excl"* ]]; then skip=1; break; fi
      done
      [[ "$skip" -eq 1 ]] && continue
      tip_sha="${range##*..}"
      if ! git cat-file -e "${tip_sha}:${file}" 2>/dev/null; then continue; fi
      content=$(git show "${tip_sha}:${file}" 2>/dev/null || true)
      for pattern in "${PATTERNS[@]}"; do
        if printf '%s' "$content" | grep -qE "$pattern"; then
          printf '%b\n' "${RED}credential-prefix match in ${file}:${NC} pattern ${pattern}" >&2
          hit=$((hit + 1))
          hits_log="${hits_log}${file}:${pattern} "
        fi
      done
    done
  done
fi

# Build artifact JSON.
exit_code=0
if [ "$hit" -gt 0 ]; then
  exit_code=1
  asserts_passed='[]'
  asserts_failed='["'"$hit"' credential-prefix match(es) on outgoing ref-range"]'
else
  asserts_passed='["no credential-prefix matches in '"${#scan_refs[@]}"' ref range(s) -- patterns set: '"${#PATTERNS[@]}"'"]'
  asserts_failed='[]'
fi

cat > "$ARTIFACT" <<EOF
{
  "ts": "$ts",
  "row_id": "$row_id",
  "exit_code": $exit_code,
  "ranges_scanned": ${#scan_refs[@]},
  "patterns_count": ${#PATTERNS[@]},
  "hits": $hit,
  "asserts_passed": $asserts_passed,
  "asserts_failed": $asserts_failed
}
EOF

exit "$exit_code"
```

The above is ~110 lines. Within the 150 cap. Anti-bloat: do NOT add new patterns; do NOT change EXCLUDE_DIRS list; do NOT refactor the scan loop. Logic is preserved verbatim from the existing hook body.
