**Why mkdocs auto-include works:** mkdocs-material's default behavior is to copy every file under the configured `docs_dir` (default: `docs/`) into the built `site/` directory. JSON / images / CSS / JS are all auto-included. The `not_in_nav: |` directive in `mkdocs.yml` (already at line 47 per the existing config) is the suppress-from-navigation mechanism but does NOT exclude files from publish -- they're still auto-included. So `docs/badge.json` becomes `site/badge.json` -> `https://reubenjohn.github.io/reposix/badge.json` after a GH Pages deploy.
