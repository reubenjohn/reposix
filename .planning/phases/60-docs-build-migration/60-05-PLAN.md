**Why bifurcate instead of having the runner pipe stdin?** Because then the runner cadence semantics get murky: weekly cron has no stdin; post-release has no stdin. Only pre-push has git ref-range stdin. The runner SHOULD NOT carry stdin-shape special-casing; that's exactly the kind of bloat Wave E's anti-bloat protects against. The hook is the right home for stdin handling; the runner is the right home for catalog dispatch.
