**rubrics/cold-reader-hero-clarity.md (verbatim prompt body, <=80 LOC):**

```markdown
You are reviewing one or more files in complete isolation. You have NOT seen the repo, the codebase, or any other files. Do not follow any links. Do not request additional context. Do not use any tools. Read only what is provided.

Your job: give honest, unbiased feedback on the document's hero section (typically the first 50 lines: title, tagline, install command, opening pitch) for a reader arriving cold — an LLM agent, a contributor, or a user who just found this file.

The hero is the entry point. It must answer three questions in 30 seconds of reading:
1. What does this software do?
2. Why would I use it?
3. How do I install or try it?

Report the following sections:

## What I learned in 30 seconds

Plain language: what the software does, who would use it, and how you would try it. Cite the line numbers you read.

## Friction Points

Specific sentences, paragraphs, or terms in the hero that are confusing, ambiguous, or require outside knowledge not given in the document.

## Unanswered Questions

After reading the hero, what do you still not know?

## Install Path Clarity

Does the hero show a working install command? Does it lead with a package manager (cargo install / brew install / cargo binstall / pip install / npm install / go install — any package-manager-first ordering)? OR does it lead with `git clone && build-from-source` (which is a worse first impression for a published tool)?

## Verdict

In 2 sentences: would an agent landing on this hero cold understand what the software does and how to try it?

Rate: CLEAR / NEEDS WORK / CONFUSING

Rationale: one paragraph explaining the rating.
```

This prompt is consumed by doc-clarity-review via:
```bash
claude /doc-clarity-review --prompt .claude/skills/reposix-quality-review/rubrics/cold-reader-hero-clarity.md README.md docs/index.md
```

doc-clarity-review writes structured output to `/tmp/doc-clarity-review-XXXXXX/_feedback.md`. The "## Verdict" section ends with `Rate: CLEAR` (or NEEDS WORK / CONFUSING).

**lib/dispatch_cold_reader.sh sketch (target <=80 LOC):**

```bash
#!/usr/bin/env bash
# .claude/skills/reposix-quality-review/lib/dispatch_cold_reader.sh
# Rubric dispatcher: invokes doc-clarity-review on README.md + docs/index.md
# with the cold-reader rubric prompt; parses verdict; persists canonical artifact.
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "$0")/../../../.." && pwd)"
SKILL_DIR="$REPO_ROOT/.claude/skills/reposix-quality-review"
RUBRIC_ID="subjective/cold-reader-hero-clarity"
RUBRIC_PROMPT="$SKILL_DIR/rubrics/cold-reader-hero-clarity.md"
SOURCE_FILES=("$REPO_ROOT/README.md" "$REPO_ROOT/docs/index.md")

# Step 1: invoke doc-clarity-review (subprocess)
TMPLOG="$(mktemp /tmp/cold-reader-dispatch-XXXXXX.log)"
trap 'rm -f "$TMPLOG"' EXIT

# We pass --prompt + the file paths; doc-clarity-review handles tmp dir + claude subprocess
claude /doc-clarity-review --prompt "$RUBRIC_PROMPT" "${SOURCE_FILES[@]}" 2>&1 | tee "$TMPLOG" || {
    # Fallback: doc-clarity-review failed; emit fallback artifact
    python3 "$SKILL_DIR/lib/persist_artifact.py" \
        --rubric-id "$RUBRIC_ID" \
        --score 0 \
        --verdict CONFUSING \
        --rationale "doc-clarity-review subprocess failed; see $TMPLOG" \
        --evidence-files "README.md" "docs/index.md" \
        --dispatched-via "doc-clarity-review-failed" \
        --asserts-failed "doc-clarity-review subprocess exited non-zero"
    exit 1
}

# Step 2: parse the verdict (the doc-clarity-review _feedback.md ends with "Rate: <VERDICT>")
# doc-clarity-review prints _feedback.md path; grep last "Rate: ..." line
VERDICT="$(grep -E '^Rate: ' "$TMPLOG" | tail -1 | sed 's/^Rate: //;s/ *$//')"
case "$VERDICT" in
    CLEAR)        SCORE=10 ;;
    "NEEDS WORK") SCORE=5  ;;
    CONFUSING)    SCORE=2  ;;
    *)            SCORE=0  ;;
esac

# Step 3: extract rationale (the paragraph after "Rationale:" in the Verdict section)
RATIONALE="$(awk '/^## Verdict/,/^## /' "$TMPLOG" | grep -A 100 'Rationale:' | tail -n +2 | head -10 | tr '\n' ' ')"
[ -z "$RATIONALE" ] && RATIONALE="(no rationale extracted)"

# Step 4: persist canonical artifact (lib/persist_artifact.py is the single writer)
python3 "$SKILL_DIR/lib/persist_artifact.py" \
    --rubric-id "$RUBRIC_ID" \
    --score "$SCORE" \
    --verdict "$VERDICT" \
    --rationale "$RATIONALE" \
    --evidence-files "README.md" "docs/index.md" \
    --dispatched-via "doc-clarity-review"

# Step 5: exit code per rubric threshold (>=7 PASS / 4-6 PARTIAL / <4 FAIL)
if [ "$SCORE" -ge 7 ]; then exit 0
elif [ "$SCORE" -ge 4 ]; then exit 2
else exit 1
fi
```

**lib/persist_artifact.py CLI extension:**

Wave C's persist_artifact.py only has a Python API. Wave D adds an argparse-based CLI shim so dispatch_cold_reader.sh can call it via subprocess. ~20 LOC addition. Final persist_artifact.py is <=100 LOC (Wave C's 80 + Wave D's 20).

```python
# At the bottom of lib/persist_artifact.py, add:
def main():
    import argparse
    p = argparse.ArgumentParser()
    p.add_argument("--rubric-id", required=True)
    p.add_argument("--score", required=True, type=int)
    p.add_argument("--verdict", required=True)
    p.add_argument("--rationale", required=True)
    p.add_argument("--evidence-files", nargs="+", default=[])
    p.add_argument("--dispatched-via", required=True)
    p.add_argument("--asserts-passed", nargs="*", default=[])
    p.add_argument("--asserts-failed", nargs="*", default=[])
    args = p.parse_args()
    artifact_path = persist_artifact(
        rubric_id=args.rubric_id,
        score=args.score,
        verdict=args.verdict,
        rationale=args.rationale,
        evidence_files=args.evidence_files,
        dispatched_via=args.dispatched_via,
        asserts_passed=args.asserts_passed,
        asserts_failed=args.asserts_failed,
    )
    print(f"persisted: {artifact_path}")


if __name__ == "__main__":
    main()
```
