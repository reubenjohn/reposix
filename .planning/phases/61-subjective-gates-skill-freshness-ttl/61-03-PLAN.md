2. **`.claude/skills/reposix-quality-review/dispatch.sh`** — bash entry point that the catalog rows' `verifier.script` field references. The catalog rows say `verifier.script = ".claude/skills/reposix-quality-review/dispatch.sh"`; this is the file. Accepts `--rubric <id>` / `--all-stale` / `--force` / (no args -> usage). <=120 LOC.

3. **`.claude/skills/reposix-quality-review/lib/catalog.py`** — stdlib helper for catalog I/O. <=80 LOC.

4. **`.claude/skills/reposix-quality-review/lib/persist_artifact.py`** — stdlib helper for artifact persistence. <=80 LOC.

**Why is the skill the catalog row's verifier?** The catalog row's `verifier.script` is the path the runner invokes. For mechanical rows, it's a shell script. For subagent-graded rows, it's the dispatch entry that brokers between "the runner needs to fire a verifier" and "an unbiased Claude subagent grades the rubric." The skill is the broker. The runner runs `bash .claude/skills/reposix-quality-review/dispatch.sh --rubric subjective/cold-reader-hero-clarity`; that script reads the rubric prompt, dispatches the subagent (Path A via Task if available; Path B in-session disclosure if not), persists the artifact, exits with the right code.

**Why parallel subagent dispatch for --all-stale?** Per OP-2 (aggressive subagent delegation): subjective rubrics are rubric checks of independent prose passages. They share no state; they share no files. Running them in parallel is the textbook OP-2 case. The `cargo memory budget` rule does NOT apply because subjective rubrics are pure-prose Claude reads — no cargo, no rustc, no rust-analyzer. 3 subagents in parallel is safe.

**Why does Wave C NOT include the rubric prompts?** Each rubric has nontrivial prompt design (the cold-reader rubric integrates `doc-clarity-review`; the install-positioning rubric needs to encode the package-manager-first ordering rule from CLAUDE.md freshness invariants; the headline-numbers rubric needs to cross-reference benchmarks/ artifacts). Splitting into Waves D-E lets each rubric get focused attention without bloating Wave C past its scaffold concern. Wave C ships rubric stub files at `rubrics/<id>.md` so the file paths exist; Wave D-E fill them in.

**Why defer catalog row.status updates to the runner re-grade?** Cleanliness:
- The runner is already the single writer of `row.status` (per `run_row` semantics).
- If `persist_artifact.py` ALSO writes status, we have two writers and a race condition (Wave G runs the skill -> persist writes status=PASS -> next runner sweep re-reads the artifact -> writes status again).
- Cleaner: persist_artifact ONLY writes the artifact; the next runner sweep reads the artifact + writes status. This is exactly how the existing mechanical verifiers behave (verifier writes the artifact JSON; runner reads + sets status).

**Why does the persist_artifact.py file exist as a separate module instead of inline in dispatch.sh?**
- 80 LOC of JSON manipulation is unreadable in bash; Python is the right tool.
- Wave G + the runner re-grade both call this code path. Sharing avoids drift.
- Future rubric implementations (in Waves D-E) will import it directly.

```python
"""Catalog I/O helper for the reposix-quality-review skill."""
import json
import sys
from datetime import datetime, timezone
from pathlib import Path

# Re-use is_stale from quality/runners/run.py
REPO_ROOT = Path(__file__).resolve().parents[3]
sys.path.insert(0, str(REPO_ROOT / "quality" / "runners"))
from run import is_stale, parse_rfc3339  # noqa


def load_subjective_catalog() -> dict:
    """Load quality/catalogs/subjective-rubrics.json."""
    return json.loads((REPO_ROOT / "quality/catalogs/subjective-rubrics.json").read_text())


def find_row(catalog: dict, rubric_id: str) -> dict:
    for r in catalog["rows"]:
        if r["id"] == rubric_id:
            return r
    raise KeyError(f"rubric not found: {rubric_id}")


def stale_rows(catalog: dict, now: datetime | None = None) -> list[dict]:
    now = now or datetime.now(timezone.utc)
    out = []
    for r in catalog["rows"]:
        if r.get("last_verified") is None or is_stale(r, now):
            out.append(r)
    return out


def all_rows(catalog: dict) -> list[dict]:
    return list(catalog["rows"])
```

**lib/persist_artifact.py sketch (target <=80 LOC):**

```python
"""Artifact-write helper for the reposix-quality-review skill."""
import json
from datetime import datetime, timezone
from pathlib import Path

REPO_ROOT = Path(__file__).resolve().parents[3]
ARTIFACTS_DIR = REPO_ROOT / "quality/reports/verifications/subjective"


def now_iso() -> str:
    return datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ")


def persist_artifact(rubric_id: str, score: int, verdict: str, rationale: str,
                     evidence_files: list, dispatched_via: str,
                     asserts_passed: list, asserts_failed: list) -> Path:
    """Write the artifact JSON for a rubric. Does NOT update catalog status.

    The runner re-grade pass reads the artifact and writes status. Single-writer
    invariant matches quality/runners/run.py:run_row semantics.
    """
    slug = rubric_id.split("/", 1)[1]
    artifact_path = ARTIFACTS_DIR / f"{slug}.json"
    artifact_path.parent.mkdir(parents=True, exist_ok=True)
    payload = {
        "ts": now_iso(),
        "rubric_id": rubric_id,
        "score": score,
        "verdict": verdict,
        "rationale": rationale,
        "evidence_files": evidence_files,
        "dispatched_via": dispatched_via,
        "asserts_passed": asserts_passed,
        "asserts_failed": asserts_failed,
        "stale": False,
    }
    artifact_path.write_text(json.dumps(payload, indent=2) + "\n")
    return artifact_path
```
