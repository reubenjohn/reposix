  <action>
Write `quality/gates/structure/verifier-script-exists.sh` (bash, mirroring
`quality/gates/structure/file-size-limits.sh`'s structure: `set -euo pipefail`,
`cd "$(git rev-parse --show-toplevel)"`, collect violations, print + exit).

Logic: for each `quality/catalogs/*.json` file whose name does NOT end in `-allowlist.json`, use a
`python3 -c` invocation (real JSON parsing is required here, not grep) to: load the file; if
`data["dimension"] == "docs-alignment"`, skip this catalog entirely (per the Interfaces note
above); else for each row in `data["rows"]`, read `row.get("verifier", {}).get("script")`:
  - if falsy/None: record a MISSING-FIELD violation (`<catalog>::<row id>::(no verifier.script
    field)::this row has no verifier.script — every non-docs-alignment row requires one`)
  - else resolve the path relative to repo root and check `os.path.isfile(path)` (MISSING violation
    if not: `<catalog>::<row id>::<path>::file does not exist::author the missing verifier, or
    correct verifier.script if the file moved`) and, if the file exists, check
    `os.access(path, os.X_OK)` (NON-EXECUTABLE violation if the bit is unset: `<catalog>::<row
    id>::<path>::not executable::chmod +x <path>`).

Print each violation on its own line. Exit 1 if any violations exist (with a header naming the
total count), else print `PASS: verifier-script-exists — N rows across M catalogs, all
verifier.script paths exist and are executable` and exit 0. Write the standard JSON artifact to
`quality/reports/verifications/structure/verifier-script-exists.json`. `chmod +x` the script.

Run it for real against the ACTUAL repo catalogs. If it finds pre-existing violations: a small
handful (a few rows with a stale path or a missing `chmod +x`) should be fixed directly as part of
this task (correct the path, or `chmod +x` the file) — that IS the point of the gate, catching
exactly this class of drift before it was structurally enforced. If the violation count is large or
structural (e.g., an entire dimension's convention is broken, suggesting a design issue rather than
a few stray files), STOP and file a `.planning/milestones/v0.15.0-phases/SURPRISES-INTAKE.md` entry
describing the finding with severity + fix-sketch, rather than silently waiving this NEW gate to
force a false PASS (a freshly-authored gate carrying a waiver on day one would defeat its own
purpose).
