Architecture (for executor context):

Post-attach, the working tree has TWO remotes:

- `origin` — plain-git (the GH mirror that originally seeded the clone).
  KEEPS plain-git semantics; NOT mutated by attach.
- `<reposix-remote-name>` (default `reposix`) — reposix-equipped; the new
  push target that bus-remote machinery (P82-P83) will eventually drive.
  `extensions.partialClone` is set to THIS remote (NOT origin).

The cache:
- Lives at `resolve_cache_path(<sot-backend>, <sot-project>)` per Q1.1
  (DECIDED: derives from SoT URL, NOT from `remote.origin.url`).
- Stores OIDs (filenames + tree structure; blobs lazy on first
  materialize) — same contract as `Cache::build_from`.
- `cache_reconciliation` table is reconciliation state, NOT an audit
  table — append-only audit semantics do not apply (INSERT OR REPLACE
  on re-attach).
- `Cache::read_blob` (existing API at `crates/reposix-cache/src/builder.rs:436`)
  is the materialization seam; it already returns `Tainted<Vec<u8>>`
  (OP-2 contract preserved — no new sanitize call sites).

Reconciliation walk (DVCS-ATTACH-02 — implemented in 79-02 T03; tested in 79-03 T01):

| Local file                         | Backend record               | Resolution                                                     |
|------------------------------------|------------------------------|----------------------------------------------------------------|
| `*.md` with `id: N` in frontmatter | record `id: N` exists        | match; row in `cache_reconciliation`                           |
| `*.md` with `id: N`                | no record `id: N`            | warn; skip; offer `--orphan-policy={delete-local,fork-as-new,abort}` |
| `*.md` with no `id` field          | n/a                          | warn; skip; not a reposix-managed file                         |
| Two local files claim `id: N`      | n/a                          | hard error (exit non-zero with both file paths in stderr)      |
| Backend record id=N                | no local file                | normal; cache marks for next fetch                             |

Re-attach semantics (DVCS-ATTACH-03 — implemented in 79-02 T02; tested in 79-03 T02):

- Same SoT spec → IDEMPOTENT (Q1.3): refreshes cache state against the
  current backend; updates `cache_reconciliation` rows; no special-casing
  of init-vs-attach origins.
- Different SoT spec → REJECTED (Q1.2): clear error citing the existing
  attached SoT.

```rust
//! to a SoT backend (DVCS-ATTACH-01..04, v0.13.0).
//!
//! Pre-conditions: CWD (or `<path>` if passed) is a git working tree

//!   1. Resolve cache path from SoT spec (NOT from `remote.origin.url`).

    DeleteLocal,

/// Run `reposix attach`. See module docs.
///
/// # Errors
/// Returns an error on any of:
/// - Spec parse failure (delegated to `translate_spec_to_url`).
/// - Working tree is not a git repo.
/// - Re-attach with a different SoT (Q1.2 reject).
/// - Duplicate `id` across two local files (architecture-sketch row 4).
/// - Cache materialization or REST list failure.
pub async fn run(args: AttachArgs) -> Result<()> {
    let spec = &args.spec;
    let work = args.path.clone().unwrap_or_else(|| PathBuf::from("."));
    if !work.join(".git").exists() {
        bail!("not a git working tree: {} (.git/ missing)", work.display());
    }

    // Step 1: derive cache path FROM SoT (Q1.1 — NOT from remote.origin.url).
    let (backend, project) = spec
        .split_once("::")
        .ok_or_else(|| anyhow!("invalid spec `{spec}`: expected `<backend>::<project>` form"))?;
    let cache_path = reposix_cache::resolve_cache_path(backend, project)
        .context("resolve cache path from SoT spec")?;

    // Q1.2 / Q1.3 — re-attach handling.
    let existing_url = git_config_get(&work, &format!("remote.{}.url", args.remote_name))?;
    if let Some(existing) = &existing_url {
        let translated = translate_spec_to_url(spec)?;
        // Strip the `?mirror=` suffix to compare core SoT spec only.
        let existing_sot = existing.split('?').next().unwrap_or(existing);
        let new_sot = translated.split('?').next().unwrap_or(&translated);
        if existing_sot != new_sot {
            bail!(
                "working tree already attached to {existing_sot}; multi-SoT not supported in v0.13.0. \
                 Run `reposix detach` first or pick the existing SoT."
            );
        }
        // Same SoT → idempotent re-attach; fall through and refresh.
    }

    // Step 2: build cache from REST.
    let mut cache = reposix_cache::Cache::open(&cache_path, backend, project)
        .context("open cache")?;
    let _tree_oid = cache.build_from().await.context("build cache from backend")?;

    // Step 3: reconcile working tree against backend (DVCS-ATTACH-02).
    let reconciliation_report = reposix_cache::reconciliation::walk_and_reconcile(
        &work,
        &mut cache,
        args.orphan_policy.into(),
    )
    .context("reconcile working tree against backend")?;

    if !reconciliation_report.duplicate_id_files.is_empty() {
        let dup = &reconciliation_report.duplicate_id_files;
        bail!(
            "duplicate id across local files: {:?}; reconciliation aborted (no rows committed)",
            dup
        );
    }

    eprintln!(
        "attach: matched={} no_id={} backend_deleted={} mirror_lag={}",
        reconciliation_report.matched_count,
        reconciliation_report.no_id_count,
        reconciliation_report.backend_deleted_count,
        reconciliation_report.mirror_lag_count,
    );

    // OP-3 audit: write a single audit_events_cache row noting attach walk completed.
    cache
        .log_attach_walk(&reconciliation_report)
        .context("write attach_walk audit row")?;

    // Step 4: compose remote URL + add (or update-if-idempotent) remote.
    let mirror_url = git_config_get(&work, &format!("remote.{}.url", args.mirror_name))?;
    let sot_url = translate_spec_to_url(spec)?;
    let remote_url = match (args.no_bus, &mirror_url) {
        (true, _) => sot_url.clone(),
        (false, Some(m)) => format!("{sot_url}?mirror={m}"),
        (false, None) => sot_url.clone(), // mirror remote not configured; user can add later
    };
    if existing_url.is_some() {
        run_git_in(&work, &["remote", "set-url", &args.remote_name, &remote_url])?;
    } else {
        run_git_in(&work, &["remote", "add", &args.remote_name, &remote_url])?;
    }

    // Step 5: set extensions.partialClone=<remote-name> (NOT origin).
    run_git_in(
        &work,
        &["config", "extensions.partialClone", &args.remote_name],
    )?;

    Ok(())
}

// Helpers `git_config_get` + `run_git_in` mirror crate::init's helpers;
// re-export from init or duplicate here. Prefer re-export to avoid drift.
fn git_config_get(work: &Path, key: &str) -> Result<Option<String>> {
    use std::process::Command;
    let out = Command::new("git").arg("-C").arg(work).args(["config", "--get", key]).output()?;
    if out.status.success() {
        Ok(Some(String::from_utf8_lossy(&out.stdout).trim().to_string()))
    } else {
        Ok(None) // git config exits 1 on missing key — not an error
    }
}

fn run_git_in(work: &Path, args: &[&str]) -> Result<()> {
    use std::process::Command;
    let out = Command::new("git").arg("-C").arg(work).args(args).output()?;
    if !out.status.success() {
        bail!("git {} failed: {}", args.join(" "), String::from_utf8_lossy(&out.stderr).trim());
    }
    Ok(())
}
```

### 3c. Walker — `crates/reposix-cache/src/reconciliation.rs`

```rust
//! Reconciliation walk: match working-tree files to backend records by
//! `id` in frontmatter (DVCS-ATTACH-02).
//!
//! Five cases per architecture-sketch.md § "Reconciliation cases":
//!   1. Match: local file's id matches a backend record → row in cache_reconciliation.
//!   2. Backend-deleted: local file's id has no matching backend record → warn + skip.
//!   3. No-id: local file lacks `id` frontmatter → warn + skip.
//!   4. Duplicate-id: two local files claim the same id → hard error.
//!   5. Mirror-lag: backend has id; no local file → cache notes for next fetch.

use std::collections::HashMap;
use std::path::{Path, PathBuf};

use crate::Cache;
use reposix_core::frontmatter;
use reposix_core::RecordId;

/// What `walk_and_reconcile` returns. Cleanly separable cases for testing
/// and for the CLI's stderr summary.
#[derive(Debug, Default)]
pub struct ReconciliationReport {
    pub matched_count: usize,
    pub no_id_count: usize,
    pub backend_deleted_count: usize,
    pub mirror_lag_count: usize,
    pub duplicate_id_files: Vec<(RecordId, Vec<PathBuf>)>,
}

/// Orphan policy controls case-2 (backend-deleted) handling.
#[derive(Debug, Clone, Copy)]
pub enum OrphanPolicy {
    Abort,
    DeleteLocal,
    ForkAsNew,
}

/// Walk the working tree, parse frontmatter, write `cache_reconciliation`
/// rows for matched records. Hard-errors on duplicate-id (case 4). Skips
/// no-id (case 3) and backend-deleted (case 2; subject to OrphanPolicy).
///
/// # Errors
/// - Returns `Err` if the SQLite write fails or duplicate-ids are
///   encountered (no rows committed in the duplicate case — atomic via
///   transaction).
/// - I/O errors during file walk are surfaced.
pub fn walk_and_reconcile(
    work: &Path,
    cache: &mut Cache,
    orphan_policy: OrphanPolicy,
) -> crate::Result<ReconciliationReport> {
    let mut local_ids: HashMap<RecordId, Vec<PathBuf>> = HashMap::new();
    let mut no_id_files: Vec<PathBuf> = Vec::new();

    for entry in walkdir::WalkDir::new(work)
        .into_iter()
        .filter_map(std::result::Result::ok)
    {
        if !entry.file_type().is_file() {
            continue;
        }
        if entry.path().extension().map_or(true, |e| e != "md") {
            continue;
        }
        // Skip files inside `.git/`.
        if entry.path().components().any(|c| c.as_os_str() == ".git") {
            continue;
        }
        let bytes = match std::fs::read_to_string(entry.path()) {
            Ok(b) => b,
            Err(_) => continue,
        };
        match frontmatter::parse(&bytes) {
            Ok(rec) => {
                local_ids.entry(rec.id).or_default().push(entry.path().to_path_buf());
            }
            Err(_) => {
                no_id_files.push(entry.path().to_path_buf());
            }
        }
    }

    let mut report = ReconciliationReport {
        no_id_count: no_id_files.len(),
        ..Default::default()
    };

    // Detect duplicates BEFORE any INSERT (atomicity).
    for (id, paths) in &local_ids {
        if paths.len() > 1 {
            report.duplicate_id_files.push((*id, paths.clone()));
        }
    }
    if !report.duplicate_id_files.is_empty() {
        // Caller handles the bail; we return early without writing.
        return Ok(report);
    }

    // Cross-reference against cache's view of backend records.
    let backend_records = cache.list_record_ids()?;
    let backend_set: std::collections::HashSet<RecordId> =
        backend_records.iter().copied().collect();

    let now = chrono::Utc::now().to_rfc3339();
    let conn = cache.connection_mut()?;
    let tx = conn.transaction()?;
    for (id, paths) in &local_ids {
        let path = &paths[0];
        if backend_set.contains(id) {
            // Case 1: match. Write cache_reconciliation row.
            // (find_oid_for_record requires &Cache, but we hold &mut conn via tx.
            //  Reorder: collect the (id, oid) pairs BEFORE the transaction.)
            // This implementation pattern: do the OID lookups first into a Vec,
            // then start the transaction, then INSERT.
            // Sketch elided; executor implements per the existing crate's
            // borrow-checker-friendly patterns.
            let oid = String::new(); // placeholder — replaced by find_oid_for_record value
            tx.execute(
                "INSERT OR REPLACE INTO cache_reconciliation (record_id, oid, local_path, attached_at) \
                 VALUES (?1, ?2, ?3, ?4)",
                rusqlite::params![id.0, oid, path.to_string_lossy(), now],
            )?;
            report.matched_count += 1;
        } else {
            // Case 2: backend-deleted. Honor orphan_policy.
            match orphan_policy {
                OrphanPolicy::Abort => {
                    eprintln!("BACKEND_DELETED id={} local_file={:?}", id.0, path);
                    report.backend_deleted_count += 1;
                }
                OrphanPolicy::DeleteLocal => {
                    let _ = std::fs::remove_file(path);
                    eprintln!("BACKEND_DELETED id={} local_file={:?} action=DELETED", id.0, path);
                    report.backend_deleted_count += 1;
                }
                OrphanPolicy::ForkAsNew => {
                    eprintln!("BACKEND_DELETED id={} local_file={:?} action=FORK_AS_NEW (TODO P82+)", id.0, path);
                    report.backend_deleted_count += 1;
                }
            }
        }
    }

    // Case 5: backend has, local lacks → mirror-lag.
    for backend_id in backend_set.iter() {
        if !local_ids.contains_key(backend_id) {
            report.mirror_lag_count += 1;
```
