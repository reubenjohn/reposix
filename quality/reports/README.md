# quality/reports/ — Output artifacts

Everything the runner and verdict generator produce lands here.
These files are committed (verdicts are the audit trail for phase closes).

## Subdirectories

### `verdicts/`

Markdown verdict reports organized by scope:

```
verdicts/
├── p<N>/              # Per-phase verdicts (phase-close gate)
│   ├── <timestamp>.md # Timestamped snapshots (multiple runs per phase)
│   └── VERDICT.md     # Final accepted verdict
├── pre-push/          # Pre-push cadence roll-ups
├── weekly/            # Weekly cadence roll-ups
├── session-end/       # End-of-session verdicts
├── milestone-v0.12.0/ # Milestone-close verdict
└── all/               # Cross-cadence roll-ups
```

Each verdict is GREEN or RED. RED on P0/P1 rows blocks phase close.

### `verifications/`

Raw per-row JSON artifacts written by gate scripts. The verdict generator
reads these to compute the roll-up. Structure mirrors dimensions:

```
verifications/<dim>/<row-slug>.json
```

### `audits/`

One-off audit reports (script retirement, cross-link analysis, repo org gaps).
Not generated by the runner — produced by subagents during specific phases.

### `doc-alignment/`

Backfill and retirement reports for the docs-alignment dimension.

## `badge.json`

Shields.io endpoint badge at `quality/reports/badge.json`. Published to the
docs site at `https://reubenjohn.github.io/reposix/badge.json`. Updated by
`verdict.py` on every run.
